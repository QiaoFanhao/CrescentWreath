# 《新月花冠》电子版术语与命名规范表 v1.1

​**文档版本**​：v1.1
​**用途**​：统一概要设计、详细设计、服务器状态模型、网络消息、代码实现中的核心术语与命名。
​**适用范围**​：当前已完成的概要设计与详细设计，以及后续所有规则、状态、消息、代码文档。

# 1. 使用规则

## 1.1 三个命名层级必须分开

本项目所有术语，统一分为三层：

### A. 中文解释名

用于文档阅读、交流说明。

### B. 规则层正式名

用于：

* 规则引擎
* 状态模型
* 时序系统
* `Effect / Action / Event` 定义
* DSL / Lua / 规则脚本语义

### C. 请求 / 消息层正式名

用于：

* 客户端请求
* 服务器响应
* 广播消息
* 网络协议
* 前后端接口定义

---

## 1.2 同一层级不能出现两个正式名

同一个概念在同一层级中，只允许有一个正式名。

例如“打出”：

* 规则层正式名只能有一个
* 请求层正式名也只能有一个

不允许写成：

* `Play / PlayTreasureCard`
* `Banish / MoveToGapZone`
* `SetAside / MoveToCharacterSetAside`

这种“二选一”形式。

---

## 1.3 实现描述不是正式术语

像：

* `MoveToGapZone`
* `MoveToCharacterSetAside`
* `MoveToOverlayContainer`

这类名字可以作为**实现说明语句**出现，
但​**不能写进术语规范表作为正式名**​。

---

## 1.4 泛词不能当正式名

像：

* 放置
* 移动
* 使用
* 处理

这些中文可以用于说明，
但如果进入正式规则层，必须落成更精确的唯一术语。

例如：

* 防御放置 → `DefensePlace`
* 特殊防御式放置 → `DefenseLikePlace`
* 盖放到角色下方 → `SetAside`
* 叠放到牌下 → `Overlay`

---

## 1.5 命名风格约束

为避免后续代码、DSL、状态机、消息层继续漂移，统一采用以下风格：

### 类型名 / 规则术语 / 事件类型 / 消息类型

使用 **PascalCase**

例如：

* `GameState`
* `ActionChain`
* `CardRevealed`
* `RevealMessage`

### 状态字段名

使用 **camelCase**

例如：

* `currentHp`
* `sigilPreview`
* `lockedSigil`
* `currentZoneId`

### 枚举值

使用 **camelCase**

例如：

* `waitingForResponse`
* `waitingForInput`
* `flow`
* `chain`

### 中文解释名

使用自然中文，不承担实现命名职责。

## 5.2 特别说明：规则层与请求层不是二选一

例如“打出”：

### 规则层

* `Play`
  表示规则语义：
  “这张牌以普通打出方式进入规则流”。

### 请求层

* `PlayTreasureCard`
  表示网络请求：
  “客户端请求服务器打出一张宝具牌”。

二者不是二选一，而是：

* ​**规则层唯一正式名**​：`Play`
* ​**请求层唯一正式名**​：`PlayTreasureCard`

同理：

* `Summon` / `SummonTreasureCard`
* `Reveal` / `RevealMessage`

---

## 6.2 资源术语冻结规则

### 魔力

* 规则层统一为：`Mana`
* 字段统一为：`mana`

### 灵符

必须严格拆分为三层，不允许混写：

* `SigilPreview` / `sigilPreview`
* `LockedSigil` / `lockedSigil`
* `IsSigilLocked` / `isSigilLocked`

#### 说明

* `sigilPreview`：行动阶段中基于阵地区推导出的预览值
* `lockedSigil`：进入召唤阶段后正式锁定、可支付的值
* `isSigilLocked`：是否已经进入正式锁定状态

> 后续不允许出现：
> 
> * `previewSigil`
> * `sigilLocked`
> * `lockedSigilValue`
> * `isLockedSigil`
>   之类变体。

### 技能值

* 规则层：`SkillPoint`
* 字段层：`skillPoint`

### 灵脉

* 规则层：`Leyline`
* 字段层：`leyline`

### 击杀分

* 规则层：`KillScore`
* 字段层：`killScore`

# 10. 事件类型命名规范

这一节新增，用于冻结事件命名风格并明确权威来源。

## 10.1 命名规则

所有事件类型统一采用 ​**PascalCase**​。

例如：

* `TurnStarted`
* `SkillDeclared`
* `DamageDeclared`
* `CardRevealed`
* `CharacterKilled`

不允许混入：

* `turn_started`
* `damage_declared`
* `damage-declared`
* `damageDeclaredEvent`

---

## 10.2 当前权威来源

当前已定义事件类型的**权威来源**为：

> 《新月花冠》电子版详细设计文档（3）
> 第 4.2 节：事件分类建议

后续若新增事件类型，必须满足：

1. 命名风格与该节保持一致
2. 先在详细设计文档中落地，再进入代码与脚本层
3. 不允许代码先发明、文档后补票

---
